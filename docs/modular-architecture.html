CCRM Modüler Mimari Rehberi

Sürdürülebilir, anlaşılır ve ölçeklenebilir JavaScript frontend yapısı

Giriş

Bu belge, CCRM projesinin frontend mimarisinin modüler bir yapıya kavuşturulması için bir rehber niteliğindedir. Karmaşık ve sürekli büyüyen bir projenin sürdürülebilirliğini sağlamak, kod tekrarını önlemek ve geliştirme süreçlerini hızlandırmak amacıyla oluşturulmuştur.

Neden Modüler Yapı?

Modüler yapı, büyük projelerde kodun daha yönetilebilir parçalara ayrılmasını sağlar. Bu sayede ekip üyeleri arasında iş bölümü kolaylaşır, bakım ve geliştirme süreçleri hızlanır, test edilebilirlik artar ve kod tekrarı azalır.

Modüler Yapının Avantajları
  • Endişelerin Ayrılması: Her modül kendi sorumluluklarını yerine getirir.
  • Yeniden Kullanılabilirlik: Bileşenler ve yardımcı fonksiyonlar çeşitli sayfalarda kullanılabilir.
  • Bakım Kolaylığı: Herhangi bir değişiklik yapmak için yalnızca ilgili modülü güncellemek yeterlidir.
  • Ölçeklenebilirlik: Proje büyüdükçe sorunsuzca yeni modüller eklenebilir.
  • Paralel Geliştirme: Ekip üyeleri aynı anda farklı modüller üzerinde çalışabilir.
  • Test Edilebilirlik: İzole modüller daha kolay ve kapsamlı test edilebilir.

Core modüller, uygulamanın temel işlevselliğini sağlayan yapı taşlarıdır. Bunlar diğer tüm modüller tarafından kullanılır ve uygulamanın genel davranışını belirler.

Örnek: api.js
// Core API modülü
import { API_URL } from './config.js';

// Gelişmiş fetch fonksiyonu
export async function fetchAPI(endpoint, options = {}) {
    const url = `${API_URL}/${endpoint.startsWith('/') ? endpoint.substring(1) : endpoint}`;
    
    // Default başlıkları ekle
    const headers = {
        'Content-Type': 'application/json',
        ...options.headers
    };
    
    try {
        // İstek gönder
        const response = await fetch(url, {
            ...options,
            headers
        });
        
        // Başarısız yanıtları kontrol et
        if (!response.ok) {
            // Hata ayrıntılarını al
            let errorMessage;
            try {
                const errorData = await response.json();
                errorMessage = errorData.error || errorData.message;
            } catch (e) {
                errorMessage = await response.text();
            }
            
            throw new Error(`API Error: ${response.status} - ${errorMessage}`);
        }
        
        // JSON yanıtı parse et ve döndür
        return await response.json();
    } catch (error) {
        console.error(`API Error for ${endpoint}:`, error);
        throw error;
    }
}

Bileşenler

Bileşenler, kullanıcı arayüzünün yeniden kullanılabilir parçalarıdır. Her bileşen, kendi görünümünü ve davranışını yönetir. Bileşenler, diğer bileşenler veya sayfa betikleri tarafından kullanılır.

Bileşen Özellikleri
  • Kapsüllenmiş - iç işleyişi gizler
  • Yapılandırılabilir - özelleştirilebilir
  • Yeniden kullanılabilir - farklı bağlamlarda kullanılabilir
  • Bağımsız - minimum dış bağımlılık
  • Sorumluluğu net - tek bir iş yapar
Yaygın Bileşenler
  • datePicker.js - Tarih seçici
  • addressSelector.js - Adres seçici/arama
  • orderTable.js - Sipariş tablosu
  • modal.js - Özelleştirilmiş modallar
  • toast.js - Bildirimler
Örnek: Modal Bileşeni
// components/modal.js
// Özelleştirilebilir modal bileşeni

export default class Modal {
    constructor(options = {}) {
        this.id = options.id || `modal-${Date.now()}`;
        this.title = options.title || '';
        this.content = options.content || '';
        this.size = options.size || 'medium'; // small, medium, large
        this.onClose = options.onClose || (() => {});
        this.element = null;
        this.bootstrapModal = null;
    }

    create() {
        // Modal zaten varsa yeniden oluşturma
        if (this.element) return this;

        // Size sınıfını belirle
        const sizeClass = {
            small: '',
            medium: 'modal-lg',
            large: 'modal-xl'
        }[this.size];

        // Modal HTML oluştur
        const modalHTML = `
            

                

                    

                        

                            
${this.title}

                            
                        

                        
${this.content}

                        

                            
                        

                    

                

            

        `;

        // Modalı DOM'a ekle
        document.body.insertAdjacentHTML('beforeend', modalHTML);
        this.element = document.getElementById(this.id);
        
        // Bootstrap modal nesnesini oluştur
        this.bootstrapModal = new bootstrap.Modal(this.element);
        
        // Kapatma olayını dinle
        this.element.addEventListener('hidden.bs.modal', () => {
            this.onClose();
        });
        
        return this;
    }
    
    // Modal içeriğini güncelle
    setContent(content) {
        if (!this.element) this.create();
        this.element.querySelector('.modal-body').innerHTML = content;
        return this;
    }
    
    // Modal başlığını güncelle
    setTitle(title) {
        if (!this.element) this.create();
        this.element.querySelector('.modal-title').textContent = title;
        return this;
    }
    
    // Footer butonları ekle
    addButton(text, type = 'secondary', clickHandler = null, options = {}) {
        if (!this.element) this.create();
        
        const btnClass = `btn btn-${type}`;
        const btn = document.createElement('button');
        btn.className = btnClass;
        btn.textContent = text;
        
        if (options.dismiss) {
            btn.dataset.bsDismiss = 'modal';
        }
        
        if (clickHandler) {
            btn.addEventListener('click', clickHandler);
        }
        
        this.element.querySelector('.modal-footer-slots').appendChild(btn);
        return this;
    }
    
    // Modal göster
    show() {
        if (!this.element) this.create();
        this.bootstrapModal.show();
        return this;
    }
    
    // Modal gizle
    hide() {
        if (this.bootstrapModal) {
            this.bootstrapModal.hide();
        }
        return this;
    }
    
    // Modalı kaldır
    destroy() {
        if (this.element) {
            this.bootstrapModal.dispose();
            this.element.remove();
            this.element = null;
            this.bootstrapModal = null;
        }
        return this;
    }
}

Servisler

Servisler, uygulamanın veri ve işlem katmanını temsil eden modüllerdir. API çağrıları, veri işleme ve iş mantığı servisler tarafından yönetilir. Servisler, kullanıcı arayüzü bileşenlerinden veri erişimini soyutlar.

Servis Sorumluluk Örnek Metodlar
orderService.js Sipariş işlemleri getOrders(), createOrder(), updateOrderStatus(), cancelOrder()
customerService.js Müşteri işlemleri findCustomer(), createCustomer(), updateCustomer(), getCustomerOrders()
productService.js Ürün işlemleri getProducts(), searchProducts(), getProductsByCategory()
addressService.js Adres işlemleri searchAddress(), saveAddress(), getCustomerAddresses()
paymentService.js Ödeme işlemleri processPayment(), getPaymentHistory(), generateInvoice()
